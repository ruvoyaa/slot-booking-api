# Working Rules

## Core Mode

- Сначала анализ и фиксация решений, потом код.
- Изменения делать минимальными и трассируемыми.
- Не добавлять сущности вне ТЗ без явной причины.
- Сохранять решение простым: один Laravel API, без лишних abstraction layers.

## Required Stages

1. Discovery и фиксация открытых вопросов.
2. Рабочее ТЗ.
3. Архитектура.
4. План реализации.
5. Реализация.
6. Тесты.
7. README.
8. Локальная проверка.
9. Review и доработка.

## Project-Specific Cautions

- Оверсел недопустим.
- Изменения `remaining` и статусов hold должны быть атомарными.
- Кеш доступности должен инвалидироваться после операций, влияющих на остатки.
- Идемпотентность должна быть детерминированной и воспроизводимой.

## Decision Policy

Если до старта кода часть ответов не получена, допускаются только минимальные безопасные предположения, и они должны быть явно записаны в README/спецификации.
