# Project Context

## Goal

Сделать минимальный Laravel API для бронирования слотов с:

- горячим кешем доступности;
- защитой от оверсела;
- транзакционной обработкой;
- идемпотентным созданием hold;
- базовым README и сценариями проверки.

## Source Inputs

Проект собран из двух исходных артефактов:

- `START PROMPT FOR CODEX`
- `task`

Оба артефакта описывают одно и то же тестовое задание:

- Laravel 12 / PHP 8.2+ / MySQL 8+;
- API доступности слотов;
- создание временного hold;
- confirm hold;
- cancel hold;
- TTL кеша 5–15 секунд;
- защита от cache stampede;
- защита от оверсела;
- README с запуском и curl-примерами.

## Expected Scale

- размер: средний;
- workflow: строгий, с обязательными стадиями анализа и review;
- архитектура: минимальная, без лишних сервисов и инфраструктуры.

## Discovery Decisions

На текущем этапе подтверждено:

- слот должен иметь `start_at` / `end_at`;
- hold может резервировать больше одного места;
- нужна отдельная модель учета `held` и `confirmed`, а не только одно поле `remaining`;
- нужна полноценная авторизация через собственную минимальную JWT-реализацию;
- первая версия запускается локально через `php artisan serve`, MySQL, `curl` и feature-тесты;
- для кеша и lock-механики используем Redis;
- по фактическому bootstrap проект поднят на Laravel 13, и для этого тестового проекта это принято как допустимое отклонение от исходного брифа Laravel 12;
- модель остатков зафиксирована через `slots.capacity`, `slots.held_count`, `slots.confirmed_count`.

## Out Of Scope By Default

- очереди;
- микросервисы;
- Docker как обязательная часть решения;
- фоновые джобы очистки expired hold;
- сложная модель ролей и ACL сверх базовой JWT-аутентификации.
