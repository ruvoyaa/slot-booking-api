# slot-booking-api

Стартовый пакет документации по проекту минимального API бронирования слотов.

Назначение:

- зафиксировать контекст и границы проекта до старта реализации;
- собрать ТЗ в рабочем виде;
- определить спорные места и базовые допущения;
- подготовить архитектурный и реализационный каркас.

Содержимое:

- `docs/project-context.md` — контекст и цель проекта;
- `docs/working-rules.md` — правила работы по этому проекту;
- `docs/spec.md` — рабочая спецификация API и модели;
- `docs/architecture.md` — минимальная архитектура решения;
- `docs/implementation-plan.md` — этапы реализации и проверки.
- `docs/project-checklist.md` — актуальный чек-лист по этапам проекта.

Статус:

- проект оценен как средний по размеру;
- workflow требуется строгий, ближе к крупному проекту;
- bootstrap завершён;
- перед Stage 3 ведём отдельный актуализируемый checklist.
